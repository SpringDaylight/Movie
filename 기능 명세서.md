# 기능 명세서

## 1) 지라 코드(이슈 키) 설계 규칙

### 프로젝트 키(예시)

- **MW** = MovieWeb

### Epic(대기능) 코드

- `MW-EPxx` 형태 (ex. MW-EP01)

### 요구사항/스토리(Functional Spec) ID 규칙 (문서/표에서 쓰는 ID)

- `MW-{MODULE}-{NNN}`
- MODULE 예시:
    - HOME, MOVIES, DETAIL, GROUP, LOG, MYPAGE, TASTE, REVIEW, NAV, UI, API, DATA
- 예시: `MW-HOME-001`, `MW-API-004`

### Jira 이슈 타입 추천

- Epic: 대기능 묶음
- Story: 사용자 가치 단위(페이지/플로우)
- Task/Sub-task: 구현 작업(API/컴포넌트/테스트)

> 팀원 5명 + 기간(1/22~3/2) 기준으로는 Epic 8~10개 + Story 25~35개 정도가 현실적입니다.
>

## 4) 지라에서 “큰 기능 → 스프린트”로 쪼개는 추천(현실적인 분할)

- **Sprint 1 (기반 구축 / P0 중심)**
    
    NAV, HOME 검색 이동, MOVIES 목록/필터, DETAIL 기본정보, LOG 작성 저장, API 기본 6개, 더미데이터
    
- **Sprint 2 (경험 완성 / P1 중심)**
    
    리뷰 상세, 댓글/좋아요, 마이페이지 토글/목록, 취향 분석 페이지
    
- **Sprint 3 (고도화 / P2)**
    
    같이정하기 고도화, 무드 스펙트럼, 예측 이유 개선, 접근성/디자인 polish

원하면 다음도 바로 만들어줄게요(추가 질문 없이 진행 가능):

1. 위 표를 **지라 임포트용 CSV 포맷**(Summary, Issue Type, Epic Link, Description, Priority)으로 변환
2. 각 요구사항별 **Acceptance Criteria(Given-When-Then)**까지 붙인 버전
3. “적합도 확률/이유”를 데모에서 구현 가능한 **간단 추천/설명 로직(규칙+가중치)** 스펙 문서
